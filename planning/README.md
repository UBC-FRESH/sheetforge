# Planning Notes

Use this directory for focused research notes, design records, and contracts that are too detailed for `ROADMAP.md`.

Planning notes should be practical and evidence-oriented:

- state the question being investigated;
- record source files, workbooks, libraries, or commands inspected;
- separate verified behavior from assumptions;
- capture unresolved risks and follow-up work;
- avoid copying private notes, raw transcripts, source workbooks, or generated bulky outputs into tracked files.
